# Pal Universe 功能规格

> 来源: docs/PRD-000-PROJECT-OVERVIEW.md + docs/epics/PRD-001..007  
> 本文件是 spec-kit 的 spec.md，供 AI Agent 阅读以理解要构建什么。

---

## 1. 一句话描述

Palworld 的超级百科——比游戏本身还精致的帕鲁数据库，搭载 AI 聊天助手。

## 2. 目标用户

- **核心用户**：Palworld 活跃玩家，查数据、规划繁殖路线
- **进阶用户**：追求完美帕鲁配种、极限配队的硬核玩家
- **新玩家**：刚入手游戏，需要引导的萌新

## 3. 功能矩阵

### P0（必须，Phase 1）

| 功能 | 描述 | 验收标准 |
|------|------|---------|
| 帕鲁图鉴 | 299 只帕鲁的完整列表+详情，含属性和技能 | AC: 全部渲染正确 |
| 帕鲁卡牌系统 | TCG 级交互卡片，悬停动效、属性对比 | AC: 60fps 动画 |
| AI 聊天助手 | 自然语言查询帕鲁/繁殖/配队/位置 | AC: 首 token < 500ms |
| 繁殖计算器 | 父代→子代/子代→父代/多代推演 | AC: 100% 匹配数据 |
| 极速搜索 | 按名称/类型/技能/工作筛选帕鲁 | AC: 500ms 内返回 |
| 自动化 CI/CD | PR → 类型检查 → 测试 → 构建 → 部署 | AC: 3min 内完成 |
| 聊天流式响应 | SSE 打字机效果 | AC: 用户可见逐字输出 |
| 移动端适配 | 手机/平板/桌面全家桶 | AC: Playwright 覆盖 |

### P1（重要，Phase 2）

| 功能 | 描述 |
|------|------|

| 队伍构建器 | 拖拽组队 + 属性克制分析 |

| 多代繁殖树图 | D3.js / React Flow 可视化 |
| 性能优化 | ISR + CDN + PWA |

### P2（锦上添花，Phase 3）

| 功能 | 描述 |
|------|------|
| 物品/武器数据库 | 1195 个物品的配方和用途 |

## 4. 核心差异化

1. **AI 聊天助手** — 4 个竞品（paldb.cc、palworld.gg、palworld.th.gl、game8）都没有
2. **TCG 卡牌 UI** — 不是表格，是交互式卡牌
3. **繁殖可视化树** — 不是黑盒计算器，是多代路线图
4. **自动化飞轮** — 从 Issue 到部署全自动
5. **中文优先** — 所有竞品都是英文

## 5. 数据模型概要

### 帕鲁
```
- 名称（英文/中文）
- 元素类型（9种）
- 工作适性（12种，带等级）
- 战斗属性（HP/ATK/DEF/速度/体力/食物）
- 主动技能（最多7个）
- 伙伴技能
- 繁殖力（用于公式计算）
- 掉落物
- 稀有度
- 分布位置
- 图片 URL
```

### 繁殖组合
```
- 父代1 + 父代2 → 子代
- 28 种特殊组合（不遵循公式）
- 繁殖力公式: floor((p1.rank + p2.rank + 1) / 2)
```

### 聊天
```
- 消息列表（role/content/timestamp）
- 意图分类（帕鲁查询/繁殖查询/配队推荐/位置查询）
- 引用数据源标记
- 快捷操作按钮
```

## 7. 不做的范围（明确排除）

- ❌ 不是游戏 Mod/工具
- ❌ 不是 PvP 模拟器
- ❌ 不是多人在线服务
- ❌ 不是商业化产品（保持免费 + 开源）
- ❌ 不做用户间的社交/评论系统（Phase 3 可重新评估）
