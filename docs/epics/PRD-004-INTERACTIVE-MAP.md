# PRD-004: 超聚合互动地图

> Status: Draft · Author: AI Agent · Updated: 2026-07-19  
> Epic Lead: TBD · Dependencies: PRD-000 (地图数据), PRD-001 (帕鲁卡片引用)

---

## 1. 概述

构建 Palworld 帕尔帕戈斯群岛的**超聚合互动地图**。四层信息叠加（帕鲁分布/资源分布/Boss 位置/快速旅行），在现有竞品中做到**图层最多、信息最全、交互最流畅**。

## 2. 用户故事

```
US-001: 我在地图上看到所有帕鲁的分布区域
US-002: 我点击某区域，看到该区域所有帕鲁的列表和刷新率
US-003: 我叠加资源图层，看到矿石/蛋/果子的位置
US-004: 我看到所有 Boss（Alpha/塔主/袭击）的位置
US-005: 我看到所有快速旅行点，点击可直接标记路线
US-006: 我搜索一只帕鲁，地图自动飞到该帕鲁分布区域并高亮
US-007: 我从 AI Chat 问"炎龙在哪"，响应中点击地图跳转直达
US-008: 我可以在手机上平滑缩放平移地图
```

## 3. 功能规格

### 3.1 图层架构

```
Layer 0: 底图
├── 岛屿地形（Palpagos Islands）
├── 海拔/水域
└── 天气效果（可选）

Layer 1: 帕鲁分布 (Pal Spawns)
├── 每只帕鲁的出现区域 (Polygon)
├── 密度热力图
├── 昼/夜切换（不同帕鲁昼夜出现不同）
└── 过滤：按属性/等级/类型

Layer 2: 资源点 (Resources)
├── 矿石 (Paldium/硫磺/石英/煤炭/铁)
├── 果子/蛋位置
├── 宝箱
└── 技能果实

Layer 3: Boss & 事件
├── Alpha Boss 位置 + 刷新倒计时
├── 塔主位置
├── 袭击位置
├── Predator 帕鲁
└── 副本入口

Layer 4: 基础设施
├── 快速旅行点
├── 商人位置
├── 孵化器
└── 据点推荐位
```

## 4. 验收标准

| ID | 标准 | 测试 |

## 5. 技术实现

- Leaflet + React-Leaflet（你项目已有）
- 瓦片图层：OpenStreetMap 或 MapTiler（暗色主题）
- GeoJSON 格式存储帕鲁分布区域
- 使用 Canvas overlay 渲染大量标注点（替代 DOM 渲染）
- 组件间通信：Zustand store（地图的 center/zoom/activeLayer）
