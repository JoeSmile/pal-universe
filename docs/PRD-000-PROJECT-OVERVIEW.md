## 1. 产品背景

### 1.1 为什么要做这个产品

Palworld（幻兽帕鲁）是一款现象级开放世界生存游戏，拥有庞大的帕鲁数据库、繁殖系统、建造系统。玩家社区对高质量数据库工具有强烈需求，但现有解决方案存在明显痛点：

| 痛点 | 说明 |
|------|------|
| ❌ 信息分散 | 帕鲁数据、繁殖组合、地图位置分散在不同站点 |
| ❌ 交互简陋 | 现有站点以表格/文字为主，UI 体验粗糙 |
| ❌ 无智能搜索 | 只能按名称搜索，无法自然语言查询 |
| ❌ 繁殖规划困难 | 现有计算器只有单次配对，无多代推演 |
| ❌ 移动端体验差 | 大部分站点未适配移动端 |

### 1.2 目标用户

- **核心用户**：Palworld 活跃玩家（日均 1h+），需要查数据、规划繁殖路线
- **进阶用户**：追求完美帕鲁配种、极限配队的硬核玩家
- **新玩家**：刚入手游戏，需要引导和FAQ的萌新
- **开发者**：关注自动化开发流程的技术爱好者

### 1.3 成功指标

| 指标 | 目标值 | 衡量方式 |
|------|--------|---------|
| 站点加载 LCP | < 1.0s | Lighthouse / Vercel Analytics |
| AI Chat 准确率 | > 90% | 用户反馈 + 人工抽检 |
| 日活用户 (DAU) | > 1000 (发布后1月) | Vercel Analytics |
| Lighthouse 评分 | 100/100/100 | 自动化 CI 检查 |
| 移动端适配 | 100% | Playwright 截屏对比 |
| 繁殖计算正确率 | 100% | 自动化测试覆盖所有组合 |

## 3. 数据模型

### 3.1 核心数据结构

```typescript
// 帕鲁
interface Pal {
  id: number
  key: string          // "001"
  name: string         // "Lamball"
  types: ElementType[] // ["neutral"]
  image: string
  suitability: WorkSuitability[]
  drops: string[]
  aura: PartnerSkill    // 伙伴技能
  description: string
  skills: ActiveSkill[]
  stats: PalStats
  breeding: BreedingData
  rarity: number
  size: 'xs' | 's' | 'm' | 'l' | 'xl'
  maps: Record<string, Coordinate[]>
}

// 繁殖组合
interface BreedingCombo {
  parent1: string
  parent2: string
  child: string
}

// 聊天消息
interface ChatMessage {
  id: string
  role: 'user' | 'assistant'
  content: string
  timestamp: number
  metadata?: {
    referencedPals?: string[]
    actions?: ChatAction[]
  }
}
```

### 3.2 数据流

```
GitHub (mlg404/palworld-paldex-api)
  │
  ├─→ fork / 子模块引用
  │
  └─→ scripts/sync-data.ts (定时同步)
        │
        ├─→ frontend/data/*.json        (原始数据缓存)
        ├─→ frontend/data/index.ts       (类型导出 + 索引)
        │
        └─→ scripts/build-knowledge.ts  (构建 RAG 知识库)
              │
              └─→ AI Chat Engine 加载
```

## 4. 非功能性需求

### 4.1 性能目标

| 指标 | 目标 | 说明 |
|------|------|------|
| FCP | < 0.5s | 首次内容绘制 |
| LCP | < 1.0s | 最大内容绘制 |
| TBT | < 50ms | 总阻塞时间 |
| CLS | < 0.1 | 累计布局偏移 |
| Lighthouse | 100/100/100 | 全满分 |
| API 响应 | < 100ms | 聊天 API 首 token |
| 构建时间 | < 2min | 全量构建 |

### 4.2 安全要求

- CSP 严格策略
- 所有用户输入 XSS 过滤
- API Rate Limiting
- 依赖 CVE 扫描 (每周)
- 代码安全扫描 (每次 PR)
- 无敏感信息硬编码

### 4.3 可维护性

- 类型覆盖 100% (strict mode)
- 组件单元测试覆盖率 > 80%
- E2E 覆盖核心用户路径
- 自动生成 changelog
- 语义化版本

## 7. 约束与风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| 游戏版本更新导致数据过时 | 高 | 中 | 自动化数据同步 + 版本标记 |
| AI API 费用超支 | 中 | 高 | Rate Limit + 缓存常见问题 + 低成本模型 |
| 竞品先做 AI Chat | 中 | 低 | 我们不只是 Chat，是完整生态 |
| 图片版权问题 | 低 | 高 | 仅用游戏内解包数据 + Wiki 公开资源 |
| Next.js 版本更新破坏兼容性 | 低 | 中 | 锁定版本 + 自动化回归测试 |
