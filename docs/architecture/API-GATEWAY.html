🔍 Schema 校验（Pydantic）

端点校验规则
/pals/search q: str(1-50)
types: Optional[list[ElementType]]
work: Optional[WorkType]
page: int(1-100), default=1
per_page: int(1-50), default=20
/pals/{name} name: str(1-100), URL encoded
/breeding/calculate parent1: str(1-100)
parent2: str(1-100)
/chat/stream messages: list[{role, content}], max=50
role: enum(user|assistant)
content: str(1-2000)
conversation_id: Optional[uuid]
/chat/messages 无需参数（按 session 自动关联）

📋 审计日志

所有非健康检查的 API 请求走审计。记录格式：
字段类型示例说明
timestampISO86012026-07-19T17:00:00+08:00请求时间
methodstringPOSTHTTP 方法
pathstring/api/v1/chat/stream请求路径
ipstring1.2.3.4来源 IP
user_agentstringMozilla/5.0...客户端标识
statusint200HTTP 状态码
duration_msint234请求耗时
tokens_usedint | null156LLM token 消耗（仅 chat）
error_codestring | nullPAL_NOT_FOUND业务错误码
存储: 本地文件轮转（按天分割，保留 7 天）→ 后续归档到对象存储
成本审计: 按 IP 维度聚合 token 消耗，超限触发告警

💰 成本熔断策略

触发条件动作
单 IP 日 token > 50000该 IP 的 /chat/stream 返回 429 BUDGET_EXCEEDED
全局日 token > 500000所有 /chat/stream 降级，返回本地匹配结果
全局月费用 > $50告警通知，人工介入
LLM 错误率 > 5%自动切换到备用模型
成本计量位置: AuditMiddleware 中捕获 response，如果是 chat 端点则记录 LLM token 消耗
存储后端: Valkey（当日）+ 审计日志文件（持久化）
