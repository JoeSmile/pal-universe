## 一、帕鲁详情页 `/pals/{name}`

### 数据来源

```
GET /api/v1/pals/{name}
→ 返回完整 JSON，包含:
  name, name_cn, elements, deck_id, rarity, size,
  stats (hp/melee_attack/shot_attack/defense/support/stamina),
  work_orders[{skill, level}],
  skills[{name, type, power, ct}],
  partner_skill[{name, description, level}],
  drops[{name, rate}],
  location[{center, region, spots}],
  breeding_rank, image
```

### 移动端适配

```
单列布局，区块垂直堆叠：
  大图 → 基本信息 → 属性条 → 技能 → 工作+繁殖
  → 掉落+位置 → AI Chat

大图缩小至 150x150，基本信息用 2 列网格
属性条宽度自适应
技能列表默认只展示前 3 个，其余折叠
```

### 交互细节

| 元素 | 交互 |
|------|------|
| 大图 | 入场 spring 动画（stiffness:100, damping:20），悬停 scale(1.05) |
| 属性条 | 动画填充（从 0 到当前值），色值按比例渐变（低红→高绿） |
| 技能列表 | 前 3 行展开，其余折叠 + "查看全部 N 个" 展开按钮 |
| 工作适性 | 图标+等级点阵，悬停显示描述 |
| 繁殖排名 | 颜色标识（<500=红稀有, >2000=蓝常见），点击跳转繁殖页 |
| 掉落物 | 按概率排序，高亮 100% 掉落 |
| 位置 | 显示坐标，点击复制；"地图查看" Phase 2 可用 |
| 底部 AI | 预填 "关于 Anubis: " 上下文，发送后跳转到 Chat 页

## 二、繁殖计算器 `/breeding`

### 使用方式

**不依赖第三方图表库。** 用现有组件体系实现：

```
React 组件:
├── pal-select.tsx         ← 帕鲁下拉选择器（搜索+选择）
├── breeding-result.tsx    ← 结果展示卡片
├── breeding-reverse.tsx   ← 反向查询结果列表
└── breeding-path.tsx      ← 多代推演路线（Phase 2 引入 React Flow）
```

### API

```
✅ GET /api/v1/breeding/calculate?parent1=X&parent2=Y
    → { parent1, parent2, child, is_special, rank_average }

✅ GET /api/v1/breeding/reverse?target=X
    → [{ parent1, parent2, is_special }]

⬜ GET /api/v1/breeding/path?start=X&target=Y&generations=5 (多代推演，待实现)
```

## 三、队伍搭配 `/team-builder`

**替代"属性相克"首页入口。**

### 功能描述

```
选择 5 只帕鲁 → 系统分析:
  - 属性覆盖度（是否有火/水/草/电等全部类型的攻击手）
  - 属性短板（被哪些类型克制？缺谁？）
  - 工作适性覆盖（5 只是否能覆盖全部工作类型？）
  - AI 推荐替换（"这只换成 Anubis 可以补地面伤害"）
```

### 页面布局

```
┌─────────────────────────────────────────────────────┐
│  ⚔️ 队伍搭配                                         │
│                                                     │
│  槽位 1    槽位 2    槽位 3    槽位 4    槽位 5       │
│ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐       │
│ │ +添加 │ │ +添加 │ │ +添加 │ │ +添加 │ │ +添加 │       │
│ └──────┘ └──────┘ └──────┘ └──────┘ └──────┘       │
│                                                     │
│  分析结果                                              │
│  ┌─────────────────────────────────────────────────┐│
│  │  ✅ 属性覆盖: 6/9                                ││
│  │  ❌ 缺失: 冰、龙、暗                              ││

│  │  ❌ 短板: 弱地面 ⚠️                               ││
│  │                                                 ││
│  │  🤖 AI 建议: 把 Anubis 换成 空涡龙 可以补龙属性   ││
│  └─────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────┘
```

### 实现方式

- 纯前端计算，不依赖后端 API
- 所有数据从 `src/data/pals.json` 本地读取
- 属性克制表从 `src/data/type-chart.json` 读取
- AI 建议走现有的 Chat 流式接口（将队伍数据作为上下文）

### 首页入口更新

`quick-links.tsx` 中把 `属性克制` 改为 `队伍搭配`：
- 图标: `LuFlame` → `LuSwords` 或 `LuShield`
- 路径: `/types` → `/team-builder`
